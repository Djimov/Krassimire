# Plano de testes inicial

## Objectivo
Validar o núcleo do MVP e os principais fluxos do sistema.

## Áreas prioritárias
- validação de datas;
- validação da bounding box;
- construção dos parâmetros de pesquisa;
- parsing da resposta STAC;
- ordenação cronológica;
- estados vazios;
- estado de comparação.

## Estratégia
- testes unitários para funções utilitárias;
- testes de integração para rotas de API;
- testes manuais guiados pelos critérios de aceitação do MVP.
