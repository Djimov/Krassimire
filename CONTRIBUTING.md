# Contributing

Este repositório segue a lógica de trabalho individual orientada pelo guia da UC.

## Regras mínimas

- commits pequenos e frequentes;
- mensagens de commit descritivas;
- actualização semanal do `docs/scope/changelog.md`;
- documentação alinhada com o código;
- não colocar credenciais reais, segredos, tokens ou ficheiros `.env` no repositório público.

## Convenção sugerida de commits

- `docs:` alterações em documentação
- `feat:` nova funcionalidade
- `fix:` correcção
- `refactor:` reorganização interna
- `test:` testes
- `chore:` tarefas de manutenção
